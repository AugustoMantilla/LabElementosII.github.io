Arduino ESP32 ATtiny85

- Resumen -

Los microcontroladores son dispositivos electrónicos integrados en un solo chip que combinan un procesador, memoria, y periféricos de entrada/salida.

La programación de microcontroladores se realiza generalmente en lenguajes como C o ensamblador, aunque también se pueden usar lenguajes como Python o Arduino que esta basado en C++.

- Introducción -

A lo largo de esta primera práctica, vamos a comparar diferentes tipos de microcontroladores, específicamente el Arduino UNO, el ESP32 y el ATtiny85. Estos serán evaluados en función de cómo se programa cada microcontrolador, es decir, si requieren un programador externo para cargar el código o no.

Además, se compararán las características técnicas de cada uno, abarcando desde su arquitectura hasta su precio en el mercado.

Por último, se probarán cuatro códigos diferentes en cada uno de los microcontroladores, lo que permitirá observar su funcionamiento y rendimiento en distintas situaciones.

- Desarrollo -

Para comenzar con esta primera práctica, es necesario descargar el Arduino IDE, ya que será la herramienta principal para escribir, compilar y cargar los códigos en los microcontroladores. El Arduino IDE es muy útil porque su lenguaje de programación principal es C++, y gracias a sus diferentes librerías integradas, podemos identificar y configurar fácilmente los microcontroladores que utilizaremos en esta práctica.

Además, estas librerías serán de gran ayuda en el futuro, ya que facilitarán la interacción entre diferentes microcontroladores y simplificarán el desarrollo de proyectos más complejos. Aquí puedes descargar el IDE

Una vez que hemos descargado el Arduino IDE, el siguiente paso es agregar las placas del ATtiny85 y del ESP32. Estas placas se pueden encontrar en los siguientes enlaces: ESP32, ATtiny85. Después de descargar las librerías correspondientes, es necesario agregarlas al Arduino IDE. Para hacer esto, debes dirigirte a File > Preferences > Additional Boards Manager URLs y pegar los enlaces que se proporcionaron anteriormente.

Programación

Arduino UNO

Para programar el Arduino UNO, necesitamos tener a mano un cable USB tipo A a USB tipo B. Por lo general, este cable viene incluido cuando compramos el Arduino UNO. Una vez que lo tenemos, conectamos la placa a un puerto USB de nuestro dispositivo (laptop).

Luego, en el Arduino IDE, seleccionamos la placa correspondiente (Arduino UNO) y el puerto al que está conectado, como se muestra en la siguiente imagen.

Por último, para cargar el código en el Arduino UNO, debemos dirigirnos a la segunda barra superior del Arduino IDE y hacer clic en el ícono de la flecha que apunta hacia la derecha, conocido como Upload. Después de hacer clic, esperamos unos segundos mientras el IDE compila y carga el código en la placa. Una vez finalizado el proceso, el Arduino comenzará a ejecutar el programa que acabamos de subir.

ESP32

Similar al Arduino UNO, para programar el ESP32 necesitamos un cable USB Tipo A a USB Tipo C (o micro USB). Este cable se utiliza para conectar la placa ESP32 a nuestro equipo (laptop) a través de uno de los puertos USB disponibles.

Siguiendo los mismos pasos que con el Arduino UNO, subimos el código al ESP32. Sin embargo, en este caso, es necesario esperar un poco más que con el Arduino UNO, ya que el proceso de carga puede tardar unos segundos adicionales debido a la mayor complejidad y capacidad del ESP32.

ATtiny85

A diferencia de los otros dos microcontroladores, el ATtiny85 no puede programarse de manera directa conectándolo con un cable USB. En este caso, es necesario utilizar un Arduino como programador. Para hacer esto, primero debemos configurar el Arduino como un ISP (In-System Programmer).

Para configurar el Arduino como ISP debemos de ir a File/Examples/ArduinoISP/ArduinoISP como se muestra en la siguiente imagen.

Después de abrir el ejemplo ArduinoISP, seleccionamos la placa Arduino UNO en el menú de herramientas. Luego, nos dirigimos a Tools/Programmer/Arduino as ISP para configurar el Arduino como programador. Una vez hecho esto, abrimos una nueva ventana en el Arduino IDE para comenzar a escribir el código principal que se cargará en el ATtiny85.

Una vez que hemos cargado el código ArduinoISP en el Arduino UNO y lo hemos configurado como programador, el siguiente paso es realizar la conexión física entre el Arduino UNO y el ATtiny85. Esta conexión es importante para poder subir el código desde el Arduino al ATtiny85.

Una vez que hemos abierto una nueva ventana en el Arduino IDE, debemos configurar correctamente las opciones para programar el ATtiny85. Estas son las configuraciones que tenemos que tener:

  • Placa: ATtiny25/45/85
  • Reloj interno: 1MHz
  • Procesador: ATtiny85
  • Programador: Arduino as ISP

Para subir el código al ATtiny85, a diferencia del ESP32 y el Arduino UNO, no utilizamos el botón estándar de Upload. En su lugar, tenemos tenemos que ir a Sketch/Upload Using Programmer para poder subir el código de manera correcta ya que si no, no podremos programar el microcontrolador.

Comparación entre microcontroladores

Característica ATtiny85 Arduino UNO ESP32
Arquitectura 8-bits 8-bits 32-bits
Frecuencias Máxima 20 MHz 16 MHz-10 MHz 80 MHz-240 MHz
Memoria Flash 8 KB 32 KB 4 MB
GPIO 6 pines 14 pines 32 pines
ADC 4 pines 6 pines 18 pines
PWM 2 canales 3 canales 16 canales
UART No ✘ Si ✔ Si ✔
I2C Si ✔ Si ✔ Si ✔
Voltaje 1.8V-5V 5V 3v-3.6v
Precio (MXN) $40-$90 $50-$200 $120-$300

Al comparar los tres microcontroladores se pudo ver que el ATtiny85 se queda limitado a comparación del Arduino UNO y el ESP32, por eso es el más económico de los tres. Por otro lado, el Arduino UNO es más versátil que su hermano ATtiny85 ya que ofrece un mejor equilibrio en sus características, que aunque no sean las mejores son bastantes decentes para proyectos de una complejidad superior. Por último, el ESP32 es el más potente de los tres y es el que tiene las mejores características, por ende es el mejor para proyectos más complejos y su capacidad de conectarse a internet lo hacen perfecto para proyectos de IoT y robótica.

Códigos

Conociendo ya las características de los microcontroladores con los que trabajaremos, desarrollaremos cuatro códigos para cada uno de ellos. Para esto, utilizaremos el material mencionado anteriormente, que incluye principalmente cables, resistencias, botones y LEDs. Con estos materiales, podremos construir circuitos simples para probar el funcionamiento de los microcontroladores. Los códigos que se implementaran serán los siguientes:

  • Encendido de un LED.
  • Parpadeo del LED.
  • Control del estado del LED mediante un botón.
  • Control del estado del LED mediante un botón utilizando condicionales if/else.
